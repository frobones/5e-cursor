## Design Decisions

### Embedding vs External Data

**Decision**: Embed all treasure tables directly in `loot_generator.py`.

**Rationale**:
- Matches the pattern used in `encounter_builder.py` for XP thresholds
- Ensures feature works without requiring specific 5etools data files
- Simplifies testing and distribution
- Tables are static and unlikely to change frequently

### Dice Rolling Implementation

Use Python's `random` module with optional seeding:

```python
import random

def roll_dice(count: int, sides: int, seed: int | None = None) -> int:
    if seed is not None:
        random.seed(seed)
    return sum(random.randint(1, sides) for _ in range(count))
```

### Reference Linking Strategy

Use existing `ReferenceLinker` class:

```python
from lib.reference_linker import ReferenceLinker

linker = ReferenceLinker(index_path)
link = linker.link_item("Potion of Healing")
# Returns: "[Potion of Healing](../../books/reference/items/potion-of-healing.md)"
```
